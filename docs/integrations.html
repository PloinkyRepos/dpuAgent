Design Specifications

Ploinky and Explorer Integration

dpuAgent is consumed through Ploinky MCP routing and used by Explorer confidential UX flows as backend authority for secure content.

Ploinky Runtime Integration

  • Agent lifecycle and env exposure are declared in dpuAgent/manifest.json.
  • MCP tool surface is declared in dpuAgent/mcp-config.json.
  • AgentServer executes DPU tools and returns actor-filtered responses to clients.

Explorer Integration Scope

Explorer confidential views consume DPU APIs for virtual roots such as /Confidential/My Space, /Confidential/Shared, and /Confidential/Secrets. Explorer UI remains presentation layer, while DPU keeps storage, ACL, and identity checks centralized.

Collaborative features such as permissions and comments are backed by DPU contracts, not by local Explorer-only state.
