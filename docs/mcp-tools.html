MCP Tools

DPU contracts are declared in dpuAgent/mcp-config.json and routed through tools/dpu_tool.sh.

Tool Groups

Group Tools Purpose
Identity and roots dpu_whoami, dpu_workspace_roots Resolve actor identity and virtual confidential roots.
Secrets lifecycle dpu_secret_list, dpu_secret_get, dpu_secret_put, dpu_secret_delete, dpu_secret_grant, dpu_secret_revoke Manage secret metadata, encrypted values, and ACLs.
Confidential objects dpu_confidential_list, dpu_confidential_get, dpu_confidential_create, dpu_confidential_update, dpu_confidential_delete Manage confidential file/folder metadata and encrypted blob content.
Comments and collaboration dpu_confidential_comment_add, dpu_confidential_comment_delete, dpu_confidential_grant, dpu_confidential_revoke Support object comments and collaboration ACL flows.
Permission check dpu_access_check Evaluate actor permission on secrets or confidential objects.

Input Contract Characteristics

  • String identifiers such as key, id, principal, and role are validated before dispatch.
  • Enum constraints are enforced for fields like scope, kind, and permission.
  • Unsupported tool names fail early and return explicit errors.

Execution Characteristics

  • maxParallelTasks is set to 1 in MCP config for controlled mutation sequencing.
  • Responses are actor-filtered and do not expose raw storage internals directly.
  • Unauthorized operations fail before secret plaintext or protected content is materialized.
