Manifest and Environment

dpuAgent uses a manifest-defined runtime profile and strict env-based configuration for encryption key material, workspace roots, and storage location.

Manifest

Main runtime file: dpuAgent/manifest.json.

{
  "container": "node:20-alpine",
  "profiles": {
    "default": {
      "env": [
        "ASSISTOS_FS_ROOT",
        "WORKSPACE_ROOT",
        "DPU_WORKSPACE_ROOT",
        { "name": "DPU_DATA_ROOT", "required": false },
        { "name": "DPU_MASTER_KEY", "allowSecretsFile": false }
      ]
    }
  }
}

Core Environment Variables

Variable Usage
DPU_MASTER_KEY Required encryption root secret for protected values and confidential content.
DPU_DATA_ROOT Optional explicit storage root for DPU state and blobs.
DPU_WORKSPACE_ROOT, ASSISTOS_FS_ROOT, WORKSPACE_ROOT Workspace root resolution chain used by runtime and storage defaults.

Standalone Runtime

The standalone server entrypoint is dpuAgent/server/standalone-mcp-server.mjs with endpoints POST /mcp and GET /health. This mode reuses the same tool dispatch and storage layers.

Test Validation

Code changes should be treated as complete only after npm test passes in dpuAgent/, together with documentation updates.
