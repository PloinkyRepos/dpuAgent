Architecture

dpuAgent is a confidential storage intermediary between Explorer and encrypted DPU state. It runs under Ploinky MCP or standalone runtime mode.

Component Overview

Component Location Role
MCP contract declaration dpuAgent/mcp-config.json Defines confidential and secret tool surface and input schemas.
Wrapper entrypoint dpuAgent/tools/dpu_tool.sh Starts one Node invocation for each MCP tool call.
Tool dispatcher dpuAgent/tools/dpu_tool.mjs Parses envelope, validates args, extracts auth info, dispatches domain functions.
Domain service dpuAgent/lib/dpu-store.mjs Implements secrets, confidential objects, comments, ACL grants/revokes, and checks.
Internal storage and ACL layers dpuAgent/lib/dpu-store-internal/* Maintains encrypted persistence, locking, identity normalization, and permission manifest logic.
Explorer plugin channel dpuAgent/IDE-plugins/* Provides runtime support and comments integration components for confidential UI flows.

Storage Separation Model

DPU separates metadata, permissions, encrypted secret values, and confidential content blobs. This keeps ACL state explicit and prevents plaintext secret values from leaking into metadata registries.

.dpu-storage/

  permissions.manifest.json
  secrets.json
  blobs/
  .lock/

Intermediary Boundary

Explorer does not own confidential persistence logic. It calls MCP tools and receives actor-filtered responses. DPU decides role visibility, access checks, and serialized outputs for each actor context.

Secrets and confidential objects are domain records with dedicated semantics, not normal filesystem entities.
